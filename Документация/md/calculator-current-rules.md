# Текущие правила расчета калькулятора

Документ фиксирует рабочие правила, которые сейчас заложены в интерфейс
калькулятора фасадных кассет. Это не замена СТО, ТУ и альбомам узлов, а
практическое описание алгоритма приложения.

## 1. Входные данные

### Фасады

Для каждого типа фасада задаются:

- `Название`
- `Ширина, мм`
- `Высота, мм`
- `Количество`
- `Проемы`

Поле `Количество` используется для одинаковых фасадов. Например, для
прямоугольного здания с двумя фасадами `24000 мм` и двумя фасадами
`12000 мм` нужно заводить два типа фасадов:

- фасад `24000 x 6500`, количество `2`;
- фасад `12000 x 6500`, количество `2`.

### Углы

Поле `Количество внешних` обязательно. Если внешних углов нет, нужно
указать `0`.

Если `Количество внешних = 0`, выбор `Угловая кассета / Уголок`
неактивен, угловые кассеты не считаются и не попадают в спецификацию.

Высота углов по умолчанию равна максимальной высоте фасада, но может быть
изменена вручную.

### Кассеты

Задаются:

- тип кассеты: `КФ-1`, `КФ-2`, `КФ-3`, `КФ-4 (30)`, `КФ-4 (17)`;
- толщина: `0,7`, `1,0`, `1,2`;
- `L, мм`;
- `H, мм`;
- покрытие: `Полиэстер` или `Колор поток`.

`Полиэстер` доступен только для толщины `0,7 мм`. Для других толщин
используется `Колор поток`.

## 2. Русты

В текущей реализации используется единый расчетный руст для направления
раскладки:

| Тип кассеты | Расчетный руст |
| --- | ---: |
| `КФ-1` | `30 мм` |
| `КФ-2` | `20 мм` |
| `КФ-3` | `5 мм` |
| `КФ-4 (30)` | `30 мм` |
| `КФ-4 (17)` | `17 мм` |

Примечание: для `КФ-1` в документации есть отдельные значения
горизонтального и вертикального шва. В текущем MVP используется одно
значение для расчетной сетки, это нужно уточнить при следующей инженерной
итерации.

## 3. Стандартная и доборная кассета

Кассета считается стандартной, если хотя бы одна сторона попадает в
стандартный размер для выбранного типа:

- `L` входит в список стандартных `L`;
- или `H` входит в список стандартных `H`.

Если ни `L`, ни `H` не являются стандартными, кассета относится к
доборной позиции.

### Стандартные размеры

| Тип | Стандартные L | Стандартные H |
| --- | --- | --- |
| `КФ-1` | `572`, `1197` | `572`, `1197` |
| `КФ-2` | `555`, `1180` | `550`, `1175` |
| `КФ-3` | `580` | `580` |
| `КФ-4 (30)` | `555` | `572` |
| `КФ-4 (17)` | `580` | `598` |

Пример: `КФ-2 H 523; L 555` считается стандартной, потому что `L = 555`
является стандартной стороной.

## 4. Подбор размеров

В блоке кассет есть два взаимоисключающих режима:

- `Стандарт по длине`
- `Стандарт по высоте`

Одновременно может быть выбран только один режим, либо ни одного.

### Стандарт по длине

Калькулятор выбирает стандартную `L`, сейчас с приоритетом минимальной
стандартной стороны. Для `КФ-2` это `L = 555 мм`.

Высота `H` подбирается так, чтобы получить одинаковые ряды по высоте и
минимизировать доборные кассеты.

### Стандарт по высоте

Калькулятор выбирает стандартную `H`, а `L` рассчитывает в рамках
допустимых размеров и текущей логики подбора.

### Ручной ввод

Если пользователь вручную меняет `L` или `H`, автоматический режим
сбрасывается.

## Правило по двухуровневой П-системе

Для `П-образной двухуровневой` системы в текущей рабочей модели калькулятора:

- основной вертикальный профиль считать как `НПШ 20x80x20 1,0`;
- промежуточный вертикальный профиль также считать как `НПШ 20x80x20 1,0`;
- если длина кассеты более `800 мм`, добавляются промежуточные вертикальные профили той же марки с шагом не более `600 мм`;
- количество промежуточных профилей в одном кассетном пролете: `ceil(L / 600) - 1`, минимум `1`.

## 5. Раскладка по высоте

Для фасада высотой `Fh`, кассеты высотой `H` и руста `R` количество
полных рядов считается так:

```text
standardRows = floor((Fh + R) / (H + R))
```

Закрытая высота:

```text
covered = standardRows * H + (standardRows - 1) * R
```

Остаток:

```text
remaining = Fh - covered
```

Если остаток больше руста, добавляется доборный ряд:

```text
additionalHeight = remaining - R
```

Если остаток меньше или равен русту, отдельный доборный ряд не создается.

Пример:

```text
Fh = 6500
H = 523
R = 20

12 * 523 + 11 * 20 = 6496
остаток = 4 мм
```

Доборный ряд не создается.

### Особое правило для КФ-1

Для `КФ-1` руст формируется нахлестом элементов кассеты, поэтому раскладка не считается как простой шаг `L + R` или `H + R`.

По СТО для `КФ-1`:

```text
Lp = L - 15
Hp = H - 25
```

В калькуляторе для `КФ-1` приняты монтажные шаги:

```text
шаг по длине фасада = L - 15
шаг по высоте фасада = H - 25
```

При этом в спецификации, площади и упаковке физическими габаритами кассеты остаются `L` и `H`.

## 6. Раскладка по длине без угловых кассет

Если внешних углов нет или угловые кассеты не используются, длина фасада
разбивается на:

- стандартные колонки `L`;
- одну доборную колонку по длине, если стандартные колонки не закрывают
размер точно.

Калькулятор не должен просто брать `ceil` и выходить за габарит фасада.
Если стандартная раскладка выходит за длину, последняя кассета должна быть
доборной.

Алгоритм:

1. Берется максимально возможное количество стандартных колонок.
2. Проверяется остаток.
3. Если остаток равен нулю или меньше/равен русту, доборная колонка не
   создается.
4. Если остаток больше руста, создается доборная кассета по длине.
5. Если остаточная ширина меньше минимально допустимой, количество
   стандартных колонок уменьшается, чтобы доборная кассета вошла в
   допустимый диапазон.

## 7. Раскладка с угловыми кассетами

Если включены угловые кассеты и `Количество внешних > 0`, рядовые
кассеты считаются внутри кассетного поля между полками угловых кассет.

Для здания с несколькими фасадами и наружными углами кассетное поле
каждого фасада шире чистой длины стены на вынос соседних плоскостей:

```text
cassetteFieldWidth = facadeWidth + 2 * subsystemProjection
```

Где `facadeWidth` — размер стены по фронту фасада, а
`subsystemProjection` — вынос системы от стены до плоскости облицовки.
Этот вынос не добавляется по высоте фасада и не создает отдельный
вертикальный ряд доборных кассет. Дополнительная ширина учитывается в
расчете угловых зон и может изменить количество рядовых кассет по длине
или размер угловой полки.

Для каждого типа фасада рассчитывается:

```text
cornerWidth = (cassetteFieldWidth - rowCassetteCount * L - seamsCount * R) / 2
```

где:

```text
seamsCount = rowCassetteCount + 1
```

Расчет перебирает количество рядовых кассет по длине от максимального к
меньшему и ищет такую ширину полной полки угловой кассеты, чтобы ее
плоская видимая часть была в диапазоне:

```text
200–700 мм
```

В расчетной формуле к этому диапазону добавляется вынос подсистемы:

```text
effectiveCornerMin = 200 + subsystemProjection
effectiveCornerMax = 700 + subsystemProjection
```

В спецификации показывается плоская часть полки:

```text
L1/L2 = cornerWidth - subsystemProjection
```

## 8. Угловые кассеты

Угловая кассета имеет:

- `H` — высота кассеты;
- `L1` — первая полка;
- `L2` — вторая полка.

В спецификации размер угловой кассеты выводится так:

```text
H 523; L1 203; L2 240
```

Количество угловых кассет:

```text
cornerPieces = outsideCorners * rowsByCornerHeight
```

Высота угловых кассет раскладывается так же, как рядовые кассеты по
высоте. Если по высоте есть остаточный ряд, он должен иметь отдельный
размер `H`.

Площадь угловой кассеты считается по полной развертке двух полок:

```text
area = (L1 + L2) * H / 1_000_000
```

## 9. Спецификация кассет

В таблице спецификации выводятся:

- элемент;
- размер;
- количество, шт;
- площадь, м2;
- код позиции прайса;
- наименование;
- цена;
- сумма, руб.

Для рядовых кассет формат размера:

```text
H 523; L 555
```

Для угловых кассет:

```text
H 523; L1 203; L2 240
```

### Возможные строки рядовых кассет

- `Рядовые стандартные`
- `Рядовые доборные`
- `Рядовые доборные по длине`
- `Рядовые стандартные по высоте`
- `Рядовые доборные по высоте`
- `Рядовые доборные по длине и высоте`

Строки с нулевым количеством скрываются.

## 10. Цена и сумма

Позиция прайса подбирается по:

- типу кассеты;
- роли: стандартная, доборная, угловая;
- толщине;
- покрытию.

Если цена найдена и единица измерения `m2`, сумма считается так:

```text
sum = areaM2 * pricePerM2
```

Для рядовых кассет площадь проемов вычитается из площади кассет перед
расчетом суммы. Вычет распределяется по строкам рядовых кассет
пропорционально их площади. Количество кассет при этом остается по
раскладке фасадной сетки, потому что проемы не всегда кратны модулю
кассет и требуют проектной раскладки/подрезки.

Внизу таблицы выводится:

```text
Итого по спецификации
```

Это сумма всех строк, где удалось определить цену.

## 11. Контрольные примеры

### Пример 1. Один фасад без внешних углов

Дано:

```text
Фасад: 12000 x 6500
Количество: 1
КФ-2
L = 555
H = 523
R = 20
Внешние углы = 0
```

Ожидаемо:

- угловые кассеты не считаются;
- выбор оформления внешних углов неактивен;
- по высоте `12` рядов;
- по длине расчет не должен выходить за габарит, при необходимости
  создается доборная кассета по длине.

### Пример 2. Прямоугольное здание 24000 x 12000

Дано:

```text
Фасад А: 24000 x 6500, количество 2
Фасад Б: 12000 x 6500, количество 2
Внешние углы: 4
Оформление: угловая кассета
КФ-2
L = 555
H = 523
R = 20
Г-образная подсистема
Утеплитель 150 мм
Вентзазор 40 мм
Вынос подсистемы для угловой кассеты: 150 + 40 + 20 = 210 мм
```

Ожидаемо:

```text
Фасад А: 40 рядовых кассет по длине, 12 рядов по высоте
Фасад Б: 19 рядовых кассет по длине, 12 рядов по высоте
```

Это правило устарело: оно считало раскладку от чистой длины стены.
Актуальное правило считает кассетное поле с учетом выноса:

```text
Фасад А: кассетное поле 24000 + 210 + 210 = 24420 мм
Фасад Б: кассетное поле 12000 + 210 + 210 = 12420 мм
Фасад А: 41 рядовая кассета по длине, 12 рядов по высоте
Фасад Б: 20 рядовых кассет по длине, 12 рядов по высоте
Рядовые: 41 * 12 * 2 + 20 * 12 * 2 = 1464 шт
```

Площадь рядовых:

```text
1464 * 555 * 523 / 1_000_000 = 424,95 м2
```

Угловые:

```text
12 шт на угол * 4 угла = 48 шт
```

## 12. Крепеж, утеплитель и пленка

Для `КФ-4` принято рабочее правило:

```text
L <= 800 мм: 4 самореза на кассету
L > 800 мм: 6 саморезов на кассету
```

Мембрана считается по площади утепления с нахлестом `200 мм`.
Для рулона `1,6 м` это дает коэффициент:

```text
1,6 / (1,6 - 0,2) = 1,142857
```

Дюбели утеплителя добавляются в спецификацию:

```text
однослойное утепление: 5 шт/м2
двухслойное утепление: 10 шт/м2
```

Анкерный крепеж кронштейнов добавляется как:

```text
1 шт на 1 кронштейн
```

Если код и цена для анкеров или дюбелей не заведены в прайс, строка
остается расчетной без суммы.

## 13. Текущие ограничения MVP

- Для `КФ-1` требуется уточнить раздельную логику горизонтального и
  вертикального руста.
- Для угловых кассет используется рабочий диапазон полки `200–700 мм`.
  Его нужно подтвердить по технической документации или заменить
  справочными ограничениями по каждому типу.
- Подбор размеров является эвристикой. Он помогает выбрать рабочий
  вариант, но не заменяет проектную раскладку и проверку внешнего вида.
- Расчет подсистемы, утеплителя, фасонных элементов и крепежа должен быть
  вынесен в отдельные спецификации.
