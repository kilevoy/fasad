# База требований для калькулятора фасадных кассет и подсистемы

## Назначение

Этот документ собирает в одном месте требования и инженерные допущения,
которые уже удалось извлечь из оцифрованных PDF. Цель документа:

- задать основу для разработки калькулятора фасадных кассет;
- выделить входные параметры и расчетные сущности;
- зафиксировать правила выбора кассет и элементов подсистемы;
- отметить данные, которые нужно добрать из чертежных и пожарных документов.

Актуальные правила, уже реализованные в интерфейсе MVP, вынесены в отдельный
файл: `calculator-current-rules.md`.

## Что уже можно считать достоверной базой

### 1. Общая модель изделия

- Фасадные кассеты описаны как металлические объемные профили для наружной облицовки стен.
- Материал кассет: тонколистовая оцинкованная сталь.
- Указаны варианты покрытия: полимерное покрытие типа "Полиэстер" или порошковое покрытие.
- Базовый диапазон толщины металла кассет: от `0.7 мм` до `1.5 мм`.

### 2. Типы кассет

- Для разработки калькулятора нужно сразу закладывать линейку `КФ-1`, `КФ-2`, `КФ-3`, `КФ-4`.
- Для `КФ-1` крепеж остается видимым и расположен в вертикальном шве.
- Для `КФ-2` и `КФ-3` шов между кассетами формируется вертикальным профилем несущей подсистемы, на который крепятся кассеты.
- Для `КФ-4` нужно выделить отдельный тип в модели данных и отдельные правила:
  - геометрии кассеты;
  - формирования шва;
  - способа крепления;
  - набора доборных элементов;
  - привязки к подсистеме.

Практический вывод:

- в архитектуре нельзя “зашивать” только три типа кассет;
- тип кассеты должен быть самостоятельной сущностью или справочником с набором правил;
- по мере уточнения документов правила `КФ-4` нужно будет формализовать так же, как для остальных типов.

### 3. Размерные правила и “экономичные” размеры

Из инструкции есть прямое указание на предпочтительные размеры, которые надо
заложить в калькулятор как рекомендованные:

- Для `КФ-2`: `550 мм` или `1175 мм` по высоте, `555 мм` или `1180 мм` по ширине.
- Для `КФ-1`: `572 мм` или `1197 мм` по высоте и ширине.
- Для `КФ-3`: `580 мм` по высоте или по ширине.

Практический вывод для продукта:

- калькулятор должен уметь не только считать произвольные размеры, но и подсказывать “экономичный” вариант раскладки;
- интерфейс должен показывать, когда выбранный размер попадает в рекомендованный набор, а когда приводит к росту подрезки и отходов.

### 4. Что должен выдавать расчет кассет

Из монтажной инструкции следует, что результат расчета должен включать:

- схему монтажа кассет;
- схему выравнивающей системы;
- направление монтажа;
- типы кассет;
- размеры кассет;
- количество кассет;
- спецификацию.

В отдельный функциональный блок калькулятора стоит заложить:

- раскладку по фасаду;
- ведомость кассет;
- ведомость подсистемы;
- ведомость доборных элементов;
- предупреждения о нестандартных местах.

### 5. Направление монтажа

В инструкции есть указание, что схема должна учитывать направление монтажа,
как правило:

- слева направо;
- снизу вверх.

Это важно для генерации раскладки и нумерации изделий.

## Требования к подсистеме

### 6. От чего зависит подбор подсистемы

Из инструкции и пояснительной записки следует, что при выборе несущей
конструкции и способа крепления учитываются:

- ветровая нагрузка;
- высота здания;
- материал стены;
- тип и толщина утеплителя;
- размеры кассет.

Это означает, что без блока подсистемы калькулятор будет неполным.
Нужен связанный расчет “кассета + несущая схема”.

### 7. Воздушный зазор

В пояснительной записке есть прямое указание на воздушный зазор:

- диапазон `40–100 мм`.

Это значение надо хранить как проектный параметр системы, влияющий на:

- конструкцию подсистемы;
- глубину выноса;
- общую толщину фасадного пирога.

### 8. Варианты каркаса и профили

Из `_11_103_30.pdf` извлекаются такие ориентиры:

- для зданий до `40 м`:
  - выравнивающие кронштейны `ВКП-125x60x40`, `ВКП-200x60x40`, `ВКП-250x60x50`, толщина `1.0 мм`;
  - несущие вертикальные и горизонтальные направляющие `П 60x27x1.0 мм`;
  - для кассет фасадных также упомянут профиль `ПШ 75x20x0.7 мм`.
- для зданий выше `40 м`:
  - кронштейны `КП-200x60x40`, `ВКП-250x60x50`, толщина `1.5–2.0 мм`;
  - направляющие `П 60x27x1.2–1.5 мм`;
  - профиль для кассет `ПШ 75x20x1.0 мм`.

Практический вывод:

- высоту здания нужно вводить обязательно;
- калькулятор должен переключать допустимые типы и толщины профилей по порогу `40 м`;
- элементы подсистемы нужно описывать как справочник с условиями применимости.

### 9. Вынос и выравнивание

Из пояснительной записки:

- конструкция кронштейнов позволяет регулировать относ вертикальных направляющих от стены;
- базовый относ указан порядка `30–40 мм`;
- увеличение достигается подбором кронштейнов разной длины.

Для системы это означает:

- в модели должен быть параметр “вынос подсистемы”;
- подбор кронштейна должен опираться на требуемый вынос и неровности основания;
- нужен контроль на допустимый диапазон регулировки.

### 10. Температурные зазоры по направляющим

В пояснительной записке есть важное правило:

- учитывать температурные деформации `1.2–1.5 мм/п.м.`;
- принимать зазор порядка `10 мм`;
- вертикальные направляющие запрещено скреплять между собой без учета этого решения.

Это нужно закладывать в правила деталировки и в автоматическую генерацию подсистемы.

## Требования к утеплению и крепежу теплоизоляции

### 11. Плотность утеплителя

Из `_11_103_30.pdf`:

- основная плотность утеплителя должна быть не менее `80–100 кг/м3`;
- для внутреннего слоя допускается утеплитель плотностью от `40 кг/м3`.

### 12. Однослойное и двухслойное решение

Из текста следует:

- при толщине утеплителя порядка `100–150 мм` слой могут делить на два;
- это делается для перевязки стыков.

Калькулятор должен поддерживать:

- однослойное утепление;
- двухслойное утепление;
- разную плотность наружного и внутреннего слоя;
- автоматический пересчет крепежа по выбранной схеме.

### 13. Нормы по дюбелям для утеплителя

Из пояснительной записки:

- не менее `5 шт./м2` при однослойном размещении плит;
- в крайних и угловых зонах `6 шт./м2`;
- не менее `10 шт./м2` при двухслойном размещении;
- в крайних и угловых зонах `12 шт./м2`.

Это уже готовые расчетные правила, которые можно положить в модуль спецификации.

### 14. Влаговетрозащита

Из текста:

- угловые зоны защищают влаговетрозащитной паропроницаемой пленкой;
- ширина угловой зоны `1.2–1.5 м`;
- нахлест пленки `150–200 мм`.

Это можно использовать:

- в автоматическом расчете дополнительных материалов;
- в правилах формирования спецификации по угловым зонам.

## Что важно для модели данных калькулятора

### 15. Сущности

Минимальный набор сущностей:

- объект / проект;
- фасад;
- участок фасада;
- проем;
- кассета;
- тип кассеты;
- раскладка;
- вертикальный шов;
- горизонтальный ряд;
- кронштейн;
- направляющий профиль;
- утеплитель;
- крепеж;
- доборный элемент;
- узел примыкания.

### 16. Входные данные

Минимальные входные данные первого релиза:

- размеры фасада;
- размеры и положение проемов;
- тип кассеты: `КФ-1`, `КФ-2`, `КФ-3`;
- желаемые размеры кассеты или режим автоматического подбора;
- направление раскладки;
- высота здания;
- материал основания стены;
- наличие и толщина утепления;
- однослойное или двухслойное утепление;
- требуемый воздушный зазор;
- ветровой район или проектная ветровая нагрузка;
- тип покрытия;
- толщина металла;
- цвет и фактура, если продукт будет считать коммерческую спецификацию.

### 17. Выходные данные

Калькулятор должен уметь выдавать:

- карту раскладки кассет;
- размеры каждой кассеты;
- количество изделий по типоразмерам;
- площадь облицовки;
- площадь отходов и процент подрезки;
- спецификацию кассет;
- спецификацию профилей;
- спецификацию кронштейнов;
- спецификацию утеплителя и дюбелей;
- перечень доборных элементов;
- список инженерных предупреждений.

## Бизнес-правила, которые уже можно реализовывать

### 18. Логика подбора кассет

- Предусмотреть режим “произвольный размер”.
- Предусмотреть режим “экономичный размер”.
- При отклонении от экономичных размеров показывать предупреждение о вероятном росте отходов.
- Разные типы кассет должны иметь разные правила видимости крепежа и формирования шва.
- Все типы `КФ-1`, `КФ-2`, `КФ-3`, `КФ-4` должны обрабатываться как разные расчетные сценарии.

### 19. Логика подбора подсистемы

- Подсистема зависит от высоты здания.
- Подсистема зависит от ветровой нагрузки.
- Подсистема зависит от размеров кассеты.
- Подсистема зависит от материала стены.
- Подсистема зависит от утеплителя и требуемого выноса.

### 20. Логика спецификации утеплителя

- Для однослойного решения применять одну норму дюбелей.
- Для двухслойного решения применять повышенную норму.
- Для угловых и крайних зон использовать отдельные коэффициенты или отдельную норму.

## Что пока требует дополнительной проработки

### 21. Чертежные альбомы

По альбомам техрешений и альбому чертежей видно, что там должны быть:

- узлы примыканий;
- решения по углам;
- решения по оконным и дверным проемам;
- цокольные и парапетные узлы;
- схемы доборных элементов;
- возможно, точные правила расположения направляющих и швов.

Их нужно выделить в отдельный второй этап:

- классифицировать типовые узлы;
- связать каждый узел с условиями применения;
- определить, какие узлы влияют на автоматический расчет материалов.

### 22. Пожарный документ

Отчет по пожарной опасности нужен как минимум для:

- допустимых сочетаний облицовки и утеплителя;
- ограничений по этажности, зонам и рассечкам;
- требований к негорючим материалам и узлам.

Этот блок надо разобрать отдельно и перевести в набор проверок соответствия.

## Рекомендуемая структура разработки

### Этап 1. Геометрия и раскладка кассет

- Модель фасада и проемов.
- Автоподбор типоразмеров кассет.
- Расчет количества кассет.
- Выделение нестандартных элементов.
- Поддержка типов `КФ-1`, `КФ-2`, `КФ-3`, `КФ-4`.

### Этап 2. Подсистема

- Подбор кронштейнов и профилей.
- Учет высоты здания, выноса, ветровой нагрузки.
- Формирование спецификации профилей и кронштейнов.

### Этап 3. Утепление и сопутствующие материалы

- Слои утеплителя.
- Дюбели.
- Влаговетрозащита.
- Дополнительные материалы по зонам фасада.
- Саморезы.
- Заклепки.
- Крепеж для кассет и доборных элементов.

### Этап 4. Узлы и правила применимости

- Примыкания.
- Углы.
- Проемы.
- Парапеты, цоколи, откосы.

### Этап 5. Проверки соответствия

- Пожарные ограничения.
- Конструктивные ограничения.
- Предупреждения по нестандартным решениям.

## Обязательный полный состав расчета

Калькулятор должен считать не только облицовку, но весь комплект материалов,
необходимый для подготовки спецификации и последующего коммерческого расчета.

### 23. Что обязательно должно считаться

- Кассеты фасадные по типам `КФ-1`, `КФ-2`, `КФ-3`, `КФ-4`.
- Доборные и фасонные элементы.
- Подсистема: кронштейны, направляющие, удлинители, соединители и связанные элементы.
- Утеплитель:
  - площадь;
  - толщина;
  - количество слоев;
  - объем;
  - количество плит или упаковок.
- Пленка / мембрана:
  - площадь покрытия;
  - нахлесты;
  - запас;
  - итоговое количество рулонов или квадратных метров.
- Крепеж:
  - саморезы;
  - заклепки;
  - дюбели для утеплителя;
  - анкеры и крепеж кронштейнов;
  - крепеж доборных элементов.

### 24. Требования к расчету утеплителя

- Поддерживать однослойное и двухслойное утепление.
- Учитывать толщину каждого слоя отдельно.
- Учитывать плотность каждого слоя отдельно, если проект это требует.
- Считать площадь утепления за вычетом проемов, если это соответствует принятой методике.
- Поддерживать коэффициент запаса на подрезку и монтажные потери.
- Уметь переводить площадь и толщину в объем.
- Уметь переводить расчет в штуки или упаковки, если будет задан формат поставки.

### 25. Требования к расчету пленки / мембраны

- Считать мембрану по площади фасада или утепляемой поверхности.
- Учитывать нахлесты между полотнами.
- Учитывать нахлесты в угловых и примыкающих зонах.
- Поддерживать коэффициент технологического запаса.
- Уметь переводить итоговую площадь в рулоны после подключения справочника поставки.

### 26. Требования к расчету саморезов и заклепок

- Саморезы и заклепки не должны быть “ручным довеском”, их надо рассчитывать автоматически.
- Нормы должны зависеть от:
  - типа кассеты;
  - схемы подсистемы;
  - количества узлов крепления;
  - длины направляющих;
  - количества доборных элементов;
  - типа примыкания;
  - крайних и угловых зон, если там норма отличается.
- Нужно предусмотреть справочник норм расхода крепежа.
- Нужно предусмотреть запас на монтаж.

### 27. Архитектура цен

После инженерной части должен подключаться независимый модуль цен.
Это принципиально важно, чтобы можно было обновлять прайс без переделки формул.

Ценовой слой должен работать отдельно от количественного расчета:

- сначала считаются количества;
- затем к каждой позиции подтягивается цена;
- потом формируются суммы по позициям, разделам и общая сумма проекта.

### 28. Что должно поддерживаться в модуле цен

- Цена за штуку.
- Цена за м2.
- Цена за м.п.
- Цена за упаковку.
- Цена за рулон.
- Цена за м3.
- Цена по типу покрытия, толщине металла и типу изделия.
- Возможность добавлять коэффициент запаса, наценку и округление.

### 29. Будущая интеграция прайс-листа

Когда будет передан прайс-лист, его нужно будет привязать как минимум к таким группам:

- фасадные кассеты `КФ-1`, `КФ-2`, `КФ-3`, `КФ-4`;
- доборные элементы;
- направляющие и кронштейны;
- утеплитель;
- пленка / мембрана;
- саморезы;
- заклепки;
- анкеры и дюбели;
- прочие комплектующие.

### 30. Итоговая цель продукта

Итоговый калькулятор должен давать два результата одновременно:

- инженерную спецификацию по количеству материалов;
- коммерческий расчет по реальным ценам после загрузки прайс-листа.

- дата цен в заголовке: `12.03.2026`;
- строк номенклатуры: около `2165`;
- в строках присутствуют поля:
  - номенклатура;
  - код;
  - группа;
  - наименование группы;
  - цена;
  - единица измерения;
  - вес 1 единицы;
  - цена за 1 тонну.

### 32. Что уже подтверждено по структуре прайса

В прайсе уже есть группы, полезные для будущего калькулятора, в том числе:

- `Кассета фасадная КФ-1`;
- `Кассета фасадная КФ-2`;
- `Кассета фасадная КФ-3`;
- `Саморезы`;
- `Комплектующие`;
- профильные группы и группы подсистемы;
- номенклатура с единицами `шт`, `м2`, `п.м`, `м3`, `упак`.

Практический вывод:

- модуль цен нужно проектировать не абстрактно, а под реальный формат этой таблицы;
- привязка цены должна идти не только по названию, но и по коду номенклатуры;
- единица измерения должна храниться в модели явно, чтобы не ломать расчет стоимости.

### 33. Как правильно встроить прайс в архитектуру

Нужны как минимум две отдельные сущности:

- `РасчетнаяПозиция`
  - что посчитано;
  - количество;
  - единица измерения;
  - тип материала;
  - параметры изделия.
- `ПрайсоваяПозиция`
  - код номенклатуры;
  - наименование;
  - группа;
  - цена;
  - единица измерения;
  - вес;
  - дата прайса.

Их нельзя смешивать в одну сущность.

Правильная схема:

- сначала калькулятор считает физические объемы и количества;
- потом выполняется сопоставление с прайсовой номенклатурой;
- потом считается стоимость.

### 34. Требования к механизму сопоставления с прайсом

Нужно предусмотреть несколько уровней сопоставления:

- по точному коду номенклатуры;
- по типу изделия;
- по группе прайса;
- по толщине металла;
- по покрытию;
- по размеру или типоразмеру;
- по единице измерения.

Это особенно важно для:

- кассет;
- профилей;
- кронштейнов;
- саморезов;
- заклепок;
- утеплителя;
- пленки;
- доборных элементов.

### 35. Ограничение текущего прайса

На текущем этапе подтверждены группы `КФ-1`, `КФ-2`, `КФ-3`, но это не отменяет
необходимость поддержки `КФ-4` в расчетной модели.

Если в текущем прайсе `КФ-4` отсутствует, архитектура все равно должна:

- поддерживать расчет `КФ-4`;
- позволять добавить цены позже без изменения ядра расчета;
- корректно помечать позиции без найденной цены.
