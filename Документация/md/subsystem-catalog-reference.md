# Нормализованный справочник подсистемы

Этот файл связывает инженерные сущности подсистемы с нормализованными прайсовыми полями.

Связанные файлы:
- [subsystem-price-reference.md](C:\КАЛЬКУЛЯТОР КАССЕТ\Документация\md\subsystem-price-reference.md)
- [subsystem-catalog-schema.md](C:\КАЛЬКУЛЯТОР КАССЕТ\Документация\md\subsystem-catalog-schema.md)
- [subsystem-catalog.seed.json](C:\КАЛЬКУЛЯТОР КАССЕТ\Документация\data\subsystem-catalog.seed.json)

## 1. Что это дает калькулятору

Калькулятор должен искать цену не по сырой строке Excel, а по нормализованной сущности:

- тип элемента
- семейство
- размер
- толщина
- покрытие
- марка стали

Пример:

```json
{
  "productType": "bracket",
  "family": "kvgu",
  "widthSeries": 50,
  "length": 230,
  "thickness": 2.0,
  "coating": "galvanized",
  "steelGrade": "none"
}
```

Эта сущность должна матчиться на:
- код `118238`
- цену `34`

## 2. Базовые семейства подсистемы

| Семейство | Что это |
| --- | --- |
| `kvp` | П-образные кронштейны |
| `kvgu` | Усиленные Г-образные кронштейны |
| `npp` | П-образные направляющие |
| `npg` | Г-образные направляющие |
| `npsh` | Шляпные профили |
| `npgs` | Сигма-профили / НПГС |
| `pz` | Z-профили |
| `rivet` | Заклепки |
| `bracket_washer` | Шайба усиления кронштейна |
| `paronite_gasket` | Паронитовая терморазрывная прокладка |

## 3. Нормализованные покрытия

| Исходное название | Нормализованное значение |
| --- | --- |
| `Оцинк.` | `galvanized` |
| `Колор-поток с двух сторон` | `colorflow_2s` |
| `Колор-поток с лицевой стороны` | `colorflow_1s` |
| `П350 (Оцинк.)` | `galvanized` + `steelGrade = p350` |
| `П390 (Оцинк.)` | `galvanized` + `steelGrade = p390` |

## 4. Правило матчинга

### 4.1. Кронштейны КВП

Матчить по полям:
- `productType`
- `family`
- `size`
- `coating`
- `steelGrade`

Пример:
- `kvp + 200 + galvanized` -> `118267`

### 4.2. Кронштейны КВГУ

Матчить по полям:
- `productType`
- `family`
- `widthSeries`
- `length`
- `thickness`
- `coating`
- `steelGrade`

Пример:
- `kvgu + 50 + 230 + 2.0 + galvanized` -> `118238`
- `kvgu + 50 + 230 + 2.0 + colorflow_2s` -> `118195`

### 4.3. Направляющие НПП

Матчить по полям:
- `family`
- `width`
- `height`
- `thickness`
- `coating`

Пример:
- `npp + 60 + 27 + 1.0 + galvanized` -> `117900`

### 4.4. Направляющие НПГ

Матчить по полям:
- `family`
- `leg`
- `thickness`
- `coating`

Пример:
- `npg + 50 + 1.0 + galvanized` -> `118128`

### 4.5. НПШ

Матчить по полям:
- `family`
- `profileType`
- `dimensions`
- `thickness`
- `coating`

Отдельный случай:
- `НПШ20 0,7х3м` считать как piece-позицию
- у него `unit = pcs`, а не `lm`

### 4.6. НПГС / ПГС

Матчить по полям:
- `family`
- `dimensions`
- `perforation`
- `thickness`
- `coating`
- `steelGrade`

### 4.7. PZ / Z

Матчить по полям:
- `family`
- `dimensions`
- `perforation`
- `thickness`
- `coating`

## 5. Статусы записей

| Статус | Смысл |
| --- | --- |
| `active` | позиция найдена в прайсе и может использоваться |
| `needs_price` | позиция нужна калькулятору, но код/цена еще не подтверждены |
| `temporary` | временная позиция |
| `do_not_use` | позиция в прайсе есть, но использовать нельзя |

## 6. Что уже можно брать в код без ручной расшифровки

Уже готовы для первой версии:
- `КВП`
- `КВГУ`
- `НПП`
- `НПГ`
- часть `НПШ`
- часть `НПГС`
- часть `PZ`
- заклепки
- шайба усиления
- паронитовая прокладка как инженерная сущность со статусом `needs_price`

## 7. Что еще надо добрать

Пока как явные позиции не подтверждены:
- `ССП`
- термопрокладки / паронитовые прокладки как прайсовые позиции
- анкеры / дюбели подсистемы
- часть `НПШ 20х80х20`

Их пока правильно держать в инженерном справочнике со статусом:
- `needs_price`

## 8. Практическое правило для импорта нового прайса

При обновлении Excel:

1. распарсить строку в нормализованные поля;
2. искать полное совпадение по ключевым полям семейства;
3. если совпадение найдено, обновлять:
- `code`
- `price`
- `weight`
- `status`;
4. если совпадение не найдено:
- не подставлять “похожую” строку;
- ставить `status = needs_price`.

## 9. Минимальный обязательный ключ поиска

### Для `kvgu`
`family + widthSeries + length + thickness + coating + steelGrade`

### Для `npp`
`family + width + height + thickness + coating`

### Для `npg`
`family + leg + thickness + coating`

### Для `npsh`
`family + profileType + dimensions + thickness + coating`

### Для `npgs`
`family + dimensions + perforation + thickness + coating + steelGrade`

### Для `pz`
`family + dimensions + perforation + thickness + coating`

### Для `paronite_gasket`
`family + areaMm2 + thickness`

## 10. Обязательные элементы, которые нельзя терять в подсистеме

- `Шайба усиления кронштейна Ш50`
  - подтверждена прайсом;
  - код `8284` — оцинкованная;
  - код `40926` — `Колор-поток с двух сторон`.
- `Паронитовая терморазрывная прокладка`
  - подтверждена документами как обязательная под кронштейн;
  - для `Г-образной` системы явно указана как прокладка из паронита;
  - площадь `3000 мм²`;
  - толщина `4 мм`;
  - пока без подтвержденного кода и цены, поэтому хранится со статусом `needs_price`.
