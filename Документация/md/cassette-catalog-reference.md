# Нормализованный Справочник Основных Кассет

Этот файл связывает инженерные правила по кассетам с нормализованными прайсовыми позициями.

Связанные файлы:
- [cassette-rules.md](C:\КАЛЬКУЛЯТОР КАССЕТ\Документация\md\cassette-rules.md)
- [cassette-catalog.seed.json](C:\КАЛЬКУЛЯТОР КАССЕТ\Документация\data\cassette-catalog.seed.json)

## 1. Базовые семейства

| Семейство | Что это |
| --- | --- |
| `kf1` | Кассета фасадная 1 |
| `kf2` | Кассета фасадная 2 |
| `kf3` | Кассета фасадная 3 |
| `kf4_17` | Кассета фасадная 4/17 |
| `kf4_30` | Кассета фасадная 4/30 |

## 2. Роли кассет

| `role` | Смысл |
| --- | --- |
| `standard` | рядовая кассета без слова `доборная` |
| `additional` | позиция со словом `доборная` |

Угловые кассеты сознательно не включены в этот seed, потому что для них будет отдельный каталог.

## 3. Ключевые поля

```json
{
  "productType": "cassette",
  "family": "kf1 | kf2 | kf3 | kf4_17 | kf4_30",

```

## 4. Нормализованные покрытия

| Исходное название | Нормализованное значение |
| --- | --- |
| без доп. пометки | `polyester` |
| `Колор-поток` | `colorflow_1s` |
| `Колор-поток с двух сторон` | `colorflow_2s` |

## 5. Стандартные стороны по типам

| Семейство | Стандартные стороны |
| --- | --- |
| `kf1` | `572`, `1197` |
| `kf2` | `555`, `550`, `1180`, `1175` |
| `kf3` | `580` |
| `kf4_17` | `580`, `598` |
| `kf4_30` | `555`, `572` |

Это инженерные поля. Они не зависят от конкретной строки прайса, но нужны для логики стандартности в калькуляторе.

## 6. Правило матчинга по прайсу

### Для основных кассет

Искать по полям:
- `productType`
- `family`
- `role`
- `thickness`
- `coating`

Примеры:
- `kf1 + standard + 0.7 + polyester` -> `2182`
- `kf2 + additional + 1.2 + colorflow_1s` -> `68978`
- `kf4_30 + standard + 1.2 + colorflow_1s` -> `71733`

## 7. Что уже есть в seed

Уже вынесены:
- рядовые позиции по `КФ1`
- рядовые позиции по `КФ2`
- рядовые позиции по `КФ3`
- рядовые позиции по `КФ4/17`
- рядовые позиции по `КФ4/30`
- доборные позиции по тем же семействам

Покрыты как минимум:
- `0.7`
- `1.2`
- `polyester`
- `colorflow_1s`
- частично `colorflow_2s`

## 8. Что пока не включено

Пока сознательно не включены:
- угловые кассеты;
- горизонтальные угловые кассеты;
- все специальные варианты вроде `Колор-тупик`;
- все размерно-специальные позиции `0,5 (H...xL...)` как отдельный подкаталог.

Для `КФ3` позиции `0,5` с явным размером в названии существуют и могут потребовать отдельного режима каталога.

## 9. Практический вывод

Теперь основные кассеты уже можно:
- считать отдельно от цены;
- матчить на реальные прайсовые коды;
- разделять на рядовые и доборные;
- связывать с инженерным правилом стандартности по сторонам `L/H`.
